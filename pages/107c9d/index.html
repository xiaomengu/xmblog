# 1、publicPath打包路径和路由模式

vue项目完成之后需要打包上线，如果不加配置直接运行 npm run build 生成dist目录，点击dist目录下的index.html文件时出现问题，根本找不到页面

打包路径：

在配置文件vue.config.js文件中

//vue.config.js
module.exports = {
    publicPath: './'
}

# 2、proxy代理和环境变量

如果你的前端应用和后端 API 服务器没有运行在同一个主机上，你需要在开发环境下将 API 请求代理到 API 服务器。这个问题可以通过 vue.config.js 中的 devServer.proxy 选项来配置。

代理配置：接口地址配置

在实际开发过程中，前后端分离必定会存在跨域，设置代理解决这个问题。

在配置文件vue.config.js文件中

//vue.config.js

module.exports = {
    devServer:  {
        // 统一设置代理
        //devServer.proxy 可以是一个指向开发环境 API 服务器的字符串：
    	proxy: 'http://localhost:3000'
    }
}

# 3、productionSourceMap映射文件

productionSourceMap 是否包含map映射文件

默认为true，设置为false后就不会存在map映射文件；用于追踪线上错误信息，没什么太大的作用，因为项目有问题是不会上线。

const path = require("path");
module.exports = {
    productionSourceMap: false
}

outputDir 输出的文件夹名称

默认输出文件夹为dist，我们也可以去修改他。

# 5、assetsDir静态资源在同一个文件夹

  • Type: string

  • Default: ''

    放置生成的静态资源 (js、css、img、fonts) 的 (相对于 outputDir 的) 目录。

assetsDir 设置静态资源也可以在同一个文件夹

// vue.config.js
module.exports = {
	assetsDir: 'assets'
}

# vue--配置根目录(用@代表src目录)

vue通常会用@来代表项目的src目录，输入路径时就不用输入/src了，可以直接用@来替换

# 在Vue文件的CSS里使用

<div class="login-center-left">
    <h2><img src="@/assets/logo.png" />{{ title }}</h2>
</div>
1
2
3

注意

​ vue文件里style的css中不能使用@来获取路径，CSS loader 会把把非根路径的url解释为相对路径。

解决方案：加~前缀（~@代表根路径） 。即：加~前缀才会解释成模块路径

background: rgb(73, 123, 199) url("~@/assets/login-bg.png") left center

vue cli3及之后的配置

# 步骤1：配置jsconfig.json

baseUrl从 Vue CLI 3.3 起已弃用，请使用publicPath。 位置：项目根目录/jsconfig.json

{ 
  "compilerOptions": {
    "publicPath": "./",
    "paths": {
        "@/*": ["src/*"]
    }
  },
 
  "exclude": ["node_modules", "dist"]
}

# 步骤2：配置vue.config.js

位置：项目根目录/vue.config.js

方法一：在chainWebpack中配置

'use strict'
const path = require('path')
const resolve = dir => path.join(__dirname, dir)
 
module.exports = {
    chainWebpack: config => {
        config.resolve.alias
            .set('@', resolve('src'))
    }
}

const resolve = dir => path.join(__dirname, dir)

    resolve: {
      alias: {
        '@': resolve('src')
      }
