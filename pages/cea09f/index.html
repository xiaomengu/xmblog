Git基础知识点整理

# Git实际开发使用

# 提交和拉取

1.项目刚开始队长新建项目远程仓库

2.队员通过队长邀请或者克隆这个项目远程仓库到本地

3.在本地项目文件夹下打开Git Bush,在项目名后会出现分支名称 （第一次一般是主分支master）

队员开发者进行开发代码后提交代码：

（1）在分支下 ，这个命令是把更改的所有文件提交到本地暂存区

git add .  //提交所有更改的文件到暂存区
1

（如果指定提交的文件，则通过下边的命令）注意：这是文件的提交

git add file1 file2 file3
//或者单个提交,就是麻烦一点
git add file1
git add file2
git add file3
1
2
3
4
5

这是文件夹的提交

git add 文件夹名
1

撤销添加到暂存区的文件

  1. git reset HEAD -- . (注意最后的一个".",这条命令帮助我们一次性撤销所有放入暂存区的文件)
  2. git reset HEAD -- filename(撤销指定目标文件)
  3. git rm --cached filename(撤销指定目标文件)
  4. 以上三种方法只是把文件从暂存区中删除了（或者叫撤销了），并不会修改文件内容，也不会删除文件，在本地夹里还能找到。撤销后的文件状态是未被暂存的状态。 另外，git rm -f filename命令，也能把文件从暂存区删除，但是此命令也同时删除了本地文件，回收站中也找不到了，提醒广大同胞，慎重使用此命令来撤销暂存区的文件。

（2）然后下边这个命令是提交文件的一些说明信息,将暂存区内容添加到本地仓库中(注意：还不是远程仓库)

git commit -am"提交的一些说明信息"
1

（3）最后提交到远程仓库

git push  //如果文件夹中没有任何文件，则不会push到远程仓库
1

（4）如果发现其他队员更新代码到远程仓库master，可以通过下边的命令拉取到自己电脑上

（此时，可以通过下边的命令查看仓库当前的状态，显示有变更的文件）

git status
1

然后拉取代码

git pull
1

实际项目开发过程中，一般都是队长在Github或者Gitee上新建一个远程仓库，一般项目主分支是master

注意：我们队员进行开发项目的时候推送代码到这个仓库，一般是不直接推送到master主分支的，主要是为了以后代码修改和测试的时候出现问题影响效率和项目主分支的线上运行

这时候我们队员就要在自己的电脑上新建一个分支，是在项目主分支master下建的一个分支，这样一来代码出现问题就不影响master主分支，以后的代码提交和修改都是在自己的间的这个分支上

主分支一般队长操作以及把所有队员的分支合并到主分支master

# 新建分支

队员

（1）新建分支（在主分支master下）此时这个分支的代码和主分支的代码一模一样，但这个分支是在自己本地

git checkout -b 分支名   //注意这个分支名不带引号的英文，最好是自己的名字缩写
1

修改代码后（此时是在本地自己新建的分支上，第一次push 需要加 origin）

git add .
git commit -am"修改了一些内容"
git push //此时会报一个fatal:提示要把你这个本地的分支提交的远程仓库需要加一个命令 origin 本地新建的分支名
1
2
3
git push origin 本地新建的分支名  //注：origin是远程仓库的名字
1

（这个两个分支互不影响）

如果你想要切换回主分支master

git checkout master
1

# 合并分支

队长

队长（一般是主分支master操作者）第一次拉取队员的代码

git pull 后会提示有新的分支建立

第一步：队长想要合并分支的代码到master 则需要切换到新建的这个分支上

git checkout 队员的分支名
1

第二步：然后再执行 git pull ,就可以拿到队员写的代码

第三步：切换回master主分支

第四步：合并分支代码到master

git merge 队员的分支名
1

# 常见的冲突

队员在自己的分支下的某个功能添加或修改了一些内容，然后队员把代码push到自己远程分支上

此时队长在master分支上也在这个功能相同的地方添加或者修改了一些内容，也把代码push到master分支上了

然后队长想要合并代码

1、队长先在master上 git pull一下，保证代码最新无误

2、然后再切换到队员的分支下，执行git pull，拉取代码

3、再切换回master分支下，执行合并分支代码 git merge 队员的分支名

此时代码出现冲突

会显示冲突的内容，当前的更改 和 传入(或者合并)的更改

商议解决冲突，哪行代码需要或者不需要或者都需要（都需要的时候注意代码的前后顺序）

队长解决冲突后，再次提交远程master分支

git add .

git commit -am"说明的信息"

git push
1
2
3
4
5

队长解决冲突后，也提交到远程仓库了，此时队员需要拉取代码

1、队员在自己的分支上 git pull（养成良好的习惯，每次提交拉取前先pull一下，保证代码最新无误）

2、然后切换到master分支下 git pull ,

3、之后切换到自己的分支,合并master分支代码到自己的分支下，执行git merge master

# 回退

如果新开发一个功能，并且上线推送到远程分支了，但是这个新功能不想要了，或者出错了

想要查看之前所有的提交,执行下边的这个命令

git log  //会显示所有的提交日志信息，包括用户名，邮箱和提交的日期，以及commit时说明的信息
1

（此时会列出很多提交的日志，按 K建和J建 上下查看）

如果想回退到某个提交版本，要记住 commit后边的版本哈希序列（一般记住前五六个就行）

按 Q建 退出查看日志

执行回退

git reset --hard 版本哈希序列
1
